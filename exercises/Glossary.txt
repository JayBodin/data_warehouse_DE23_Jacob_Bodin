downstream = Mottagaren av data
upstream = Steg eller system som genererar eller skickar data vidare till nästa steg (Källan)
data warehouse = Digitalt lager där data från olika källor samlas och lagras.
cloud computing	= Använder datorresurser som lagring, databaser och servrar via internet istället för på din egen dator eller server. Hyra en data på nätet och använda den istället för din egna dator
modern data stack = En uppsättning av moderna, molnbaserade verktyg som tillsammans gör hanteringen och analysen av data. Samla data, lagring av data, bearbetning av data och visualisering av data.
idempotent = Om en operation upprepas flera gånger, kommer den inte att förändra systemet ytterligare efter den första utförda operationen. Uppdatera ett namn i en databas till "Anna", oavsett hur många förfrågot du skickar så blir den alltid samma.
OLAP = Används för att göra det enkelt att analysera stora datamängder ur olika perspektiv
OLTP = Används för att hantera och bearbeta många små transaktioner i realtid
virtual warehouse = En kraftfull, virtuell datorkluster som används för att bearbeta och köra frågor (queries) på data som lagras i ditt datalager (data warehouse).
external stage = Är en plats där du lagrar data utanför Snowflakes egen lagring, oftast i en molntjänst som Amazon S3, Microsoft Azure Blob Storage, eller Google Cloud Storage
data consumer = Är någon eller något som använder data för att göra analyser, ta beslut eller skapa rapporter
scaling out = Istället för att göra en enskild server eller dator kraftfullare lägger du till fler servrar, datorer, eller instanser i ditt system.
scaling up = Är en strategi för att öka kapaciteten hos ett system genom att uppgradera eller förstärka en enskild enhet, såsom en server eller en dator
snowflake credit = En enhet som mäter hur mycket datorkapacitet eller resurser du använder när du kör SQL-frågor eller bearbetar data. Snowflake använder credits för att prissätta sina tjänster
securable object = En enhet som du kan ställa in behörigheter för att säkerställa att endast auktoriserade användare har åtkomst till det.
snowflake object = Olika resurserna och komponenterna som du arbetar med inom Snowflake-plattformen för att hantera, bearbeta och analysera data. Tabller, vyer, scheman, databaser, roller, användare etc.
schema = Ett schema är en logisk behållare i en databas som hjälper till att organisera och hantera data och dess tillhörande objekt
permanent table	= tabell som är kvar tills den tas bort.
transient table	= En typ av tabell som är avsedd för kortvarig datalagring där data inte behöver vara permanent eller skyddad med fail-safe
temporary table	= En tabell som är tillgänglig endast under den session eller transaktion där den skapades
time-travel	= En funktion som gör det möjligt att se och återställa data till tidigare tillstånd. Det erbjuder flexibilitet för att hantera fel, granska historiska data och återställa information som har ändrats eller tagits bort. Återkalla data upp till 90 dagar.
fail-safe = Ett extra skyddslager för att återställa data i händelse av allvarliga dataförluster eller katastrofala situationer.
table = Enhet för att organisera och lagra data i en strukturerad form.
DML	= Data Manipulation Language, används för att hantera och manipulera data i databaser. SELECT, INSERT, UPDATE, och DELETE, och de tillåter användare att hämta, lägga till, ändra och ta bort data i tabeller.
DDL	= Data Definition Language, definiera och hantera databasens struktur. CREATE, ALTER, DROP, TRUNCATE, och RENAME. Dessa kommandon tillåter användare att skapa, ändra, ta bort och hantera databasobjekt såsom tabeller och vyer
DQL	= Data Query Language,  som används för att hämta och fråga data från en databas. SELECT, FROM, WHERE, ORDER, GROUP, HAVING, LIMIT.
DCL = Data Control Language, som används för att kontrollera åtkomst och behörigheter till databasobjekt. GRANT, som tilldelar behörigheter till användare eller roller, och REVOKE, som återkallar dessa behörigheter.
sysadmin = Skapa och administrera användarkonton, roller och behörigheter. installera och konfigurera programvara och hårdvara, övervaka systemprestanda, säkerställa datasäkerhet och åtgärda tekniska problem.
useradmin =  hantera användarkonton och deras behörigheter i ett IT-system eller nätverk
orgadmin = hantera hela organisationens IT-miljö, inklusive systemkonfiguration, användar- och rollhantering, säkerhet och compliance
securityadmin = tilldela och återkalla behörigheter, implementera säkerhetspolicyer, och övervaka säkerhetsrelaterade aktiviteter.
accountadmin = Detta inkluderar att skapa och hantera databaser, scheman, tabeller, användare, roller och alla andra kontoinställningar
role inheritence = roll kan ärva behörigheter och privilegier från en annan roll
public role = standardroll som automatiskt tilldelas alla användare och roller i Snowflake. Den har inga särskilda privilegier som är förknippade med den i utgångsläget.
API = Application Programming Interface. Det är en uppsättning regler och verktyg som gör det möjligt för olika mjukvaror att kommunicera med varandra De definierar hur en programvara ska begära och ta emot data från en annan.
ETL = samla in data från olika källor, bearbeta och omvandla den, och sedan ladda den till ett datalager eller en annan databas.
ELT = hämtar data från källor, laddar den till ett datalager, och sedan bearbetar och omvandlar den där
data ingestion	=  hämta och samla in data från olika källor till en central plats för vidare bearbetning och analys.
batch ingestion = samla in och bearbeta data i större grupper eller "batcher" vid specificerade tidpunkter snarare än kontinuerligt i realtid.
streamin ingestion = kontinuerligt samla in och bearbeta data i realtid
incremental load = att effektivt uppdatera data genom att endast bearbeta de poster som har förändrats sedan senaste uppdatering. Detta gör processen snabbare och mer effektiv
pagination = hantera stora mängder data genom att dela upp dem i mindre, hanterbara delar eller sidor
DLT Connectors = programvaru- eller kodkomponenter som möjliggör integration med olika datakällor och målsystem för att ladda, transformera och flytta data mellan dem.
snowflake user	= individ som har tillgång till och interagerar med Snowflake-miljön
staging layer = en säker och isolerad plats för att hantera och förbereda data innan det laddas in i en databas
granted to	= tilldelar grant till någon roll
granted on = beskriver objektet som grant blivit tilldelat till.
granted by	= den som gav grant
secrets.toml = konfigurationsfil som används för att hantera känslig information såsom autentiseringstoken, API-nycklar, databaslösenord och andra hemligheter som behövs för att ansluta till tjänster
RBAC = Role-Based Access Control, där roll styr vilken access man ska få. 
CRUD operations	= CRUD står för Create, Read, Update, och Delete, och representerar de fyra mest grundläggande operationerna som kan utföras
dlt resource = Python-funktion som används för att extrahera och förbereda data för inladdning i en datarapportpipeline
dlt source = Definiera varifrån data ska hämtas.
yield python = skapa generatorer. En generator är en speciell typ av iterator som kan pausas och återupptas, och som inte behöver lagra hela resultatet i minnet
